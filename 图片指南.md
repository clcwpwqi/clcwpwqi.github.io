# 博客图片完全指南

> 本教程介绍博客中所有图片的使用位置、格式要求及配置方法。

---

## 📋 图片要求总览

| 图片类型 | 路径 | 格式 | 尺寸 | 大小建议 | 用途 |
|---------|------|------|------|---------|------|
| **开发者头像** | `/images/avatar.png` | PNG | 200×200px | ≤100KB | 关于页面头像 |
| **站点图标** | `/images/top.png` | PNG（支持透明） | 64×64px | ≤20KB | 顶栏/底栏 Logo |
| **微信二维码** | `/images/wechat-pay.png` | PNG | 宽度≤400px | ≤200KB | 关于页微信打赏码 |
| **支付宝二维码** | `/images/alipay.jpg` | JPG/PNG | 宽度≤400px | ≤200KB | 关于页支付宝打赏码 |
| **文章封面** | `/posts/分类/文章名.png` | PNG | ≤1200×630px（16:9） | ≤300KB | 文章卡片头图 |

**⚠️ 重要：** 所有图片必须放在 `public/` 目录下（构建时会自动复制），推荐使用 `/images/` 或 `/posts/` 相对路径。

## 1️⃣ 开发者头像

### 要求
- **格式：** PNG 或 JPG
- **尺寸：** 200×200 像素（正方形）
- **大小：** 建议小于 100KB

### 配置位置
`configs/site.json` 和 `configs/about.json`：

```json
{
  "avatar": {
    "src": "/images/avatar.png",
    "width": 200,
    "height": 200
  }
}
```

### 示例
将你的头像重命名为 `avatar.png`，放入 `public/images/` 目录。

---

## 2️⃣ 站点图标（顶栏/底栏 Logo）

### 要求
- **格式：** PNG（强烈建议支持透明背景）
- **尺寸：** 64×64 像素（正方形，会自动缩放到合适大小显示）
- **大小：** 建议小于 20KB
- **背景：** 建议使用透明背景，以适配亮色和暗色主题

### 配置位置
`configs/navigation.json`（顶栏）：
```json
{
  "brand": {
    "showLogo": true,
    "logo": "C",
    "name": "clc'blog",
    "icon": "/images/top.png"
  }
}
```

`configs/footer.json`（底栏）：
```json
{
  "brand": {
    "showLogo": true,
    "logo": "C",
    "name": "clc'blog",
    "description": "一个简洁、现代的开发者博客",
    "icon": "/images/top.png"
  }
}
```

### 效果
- 如果设置了 `icon` 字段，顶栏和底栏的博客名称旁将显示该图片作为 Logo
- 如果图片加载失败，会自动降级为字母 Logo（`logo` 字段的值）
- 将 `"icon"` 设为空字符串或删除该字段，可恢复为默认字母 Logo

### 示例
将你的 Logo 图片保存为 `top.png`（64×64px，PNG 透明背景），放入 `public/images/` 目录。

## 3️⃣ 二维码/支付码

### 要求
- **格式：** PNG 或 JPG
- **尺寸：** 宽度不超过 400px
- **大小：** 建议小于 200KB

### 配置位置
`configs/about.json`：
```json
{
  "support": {
    "methods": [
      {
        "text": "微信打赏",
        "icon": "Heart",
        "image": "/images/wechat-pay.png"
      },
      {
        "text": "支付宝打赏",
        "icon": "Heart",
        "image": "/images/alipay.jpg"
      }
    ]
  }
}
```

### 示例
将微信收款码保存为 `wechat-pay.png`，支付宝收款码保存为 `alipay.jpg`（或 `.png`），放入 `public/images/` 目录。

## 4️⃣ 文章封面图片

### 要求
- **格式：** PNG
- **尺寸：** 不超过 1200×630px（16:9 比例最佳）
- **大小：** 建议小于 300KB
- **命名规则：** 必须与文章文件名（不含 `.md` 后缀）同名

### 命名规则详解

| 文章文件路径 | 封面图片路径 |
|-------------|-------------|
| `/posts/test/markdown-test.md` | `/posts/test/markdown-test.png` |
| `/posts/frontend/react-guide.md` | `/posts/frontend/react-guide.png` |
| `/posts/backend/api-design.md` | `/posts/backend/api-design.png` |

### 工作原理
1. 构建时，脚本会自动检查文章同目录下是否有同名的 `.png` 文件
2. 如果有，自动将其设为该文章的封面图片
3. 如果没有，且 frontmatter 中未指定 `cover` 字段，则使用文章首字作为默认封面
4. 如果 frontmatter 中手动指定了 `cover` 字段，则优先使用手动设置的值

### 示例
在 `posts/test/` 目录下：
```
posts/test/
├── test.json              # 分类配置
├── markdown-test.md       # 文章
└── markdown-test.png      # ← 同名封面图片（自动匹配）
```

### frontmatter 示例
```yaml
---
title: 我的文章
slug: markdown-test
date: 2024-01-01
# cover 字段可选，不设置时自动匹配同名 .png
# cover: /custom/path/image.png
---
```

## 🛠️ 自动检查与缩放

博客提供了图片检查脚本，可自动检测不符合要求的图片并进行缩放：

```bash
# 检查所有图片（仅报告，不修改）
node scripts/resize-images.js

# 自动缩放不符合要求的图片
node scripts/resize-images.js --resize
```

**依赖：** 需要安装 ImageMagick：
```bash
sudo apt install imagemagick
```

## ❓ 常见问题

### Q: 图片显示不出来？
A: 确认图片放在 `public/` 目录下（不是根目录的 `images/`），且路径以 `/` 开头。

### Q: 头像图片是 .png 但实际是 JPEG？
A: 运行 `node scripts/resize-images.js --resize` 会自动修正格式不匹配的问题。

### Q: 站点图标显示为字母而不是图片？
A: 检查 `navigation.json` 和 `footer.json` 中 `brand.icon` 字段是否正确设置，且图片路径正确。

### Q: 文章封面没有显示？
A: 确认 `.png` 图片与 `.md` 文件在同一个分类目录下，且文件名（不含后缀）完全相同。
