# 关于页面自定义教程

> 本教程介绍如何自定义关于页面（/about）的所有内容，包括"关于我"、"关于博客"、"技术栈"和"作者标签"。

---

## 📋 关于页面内容来源

关于页面的内容支持两种方式配置：

1. **Markdown 文件**（推荐）— 放在 `configs/about/` 目录下，支持 Markdown 格式
2. **JSON 配置**（兼容）— 在 `configs/about.json` 中配置，作为备用方案

**优先级：** Markdown 文件 > JSON 配置

---

## 1️⃣ 关于我

### 方式 A：Markdown 文件（推荐）

编辑 `configs/about/about-me.md`：

```markdown
---
title: "关于我"
order: 1
show: true
---

本人技术涉猎主要包括：
C++、Python、Shell、HTML、Linux 及 Android 开发

设备环境：
Windows 11 
Ubuntu 22.04
Android 16
```

**Frontmatter 说明：**

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `title` | 区块标题 | "关于我" |
| `order` | 显示顺序 | 1 |
| `show` | 是否显示（`true`/`false`） | `true` |

### 方式 B：JSON 配置（备用）

如果 `about-me.md` 不存在，会使用 `configs/about.json` 中的 `profile.bio` 字段：

```json
{
  "profile": {
    "bio": "分享技术文章和开发经验。主要擅长C++ Python HTML Shell Linux 和 Andorid 开发。"
  }
}
```

---

## 2️⃣ 关于博客

### 方式 A：Markdown 文件（推荐）

编辑 `configs/about/about-blog.md`：

```markdown
---
title: "关于博客"
order: 2
show: true
---

clc'blog 是一个基于 React + TypeScript + Tailwind CSS 构建的静态博客...

（支持任意 Markdown 格式）
```

### 方式 B：JSON 配置（备用）

```json
{
  "blog": {
    "enabled": true,
    "title": "关于博客",
    "content": "博客介绍内容..."
  }
}
```

---

## 3️⃣ 技术栈

### 方式 A：Markdown 文件（推荐）

编辑 `configs/about/tech-stack.md`：

```markdown
---
title: "技术栈"
order: 3
show: true
---

C++
Python
Node.js
HTML
Android
Docker
Git
Linux
```

**格式要求：** 每行一个技术名称，以 `#` 开头的行会被忽略。

### 方式 B：JSON 配置（备用）

```json
{
  "profile": {
    "skills": ["C++", "Python", "Node.js", "HTML", "Android", "Docker", "Git", "Linux"]
  }
}
```

---

## 4️⃣ 作者标签（新增功能）

在关于页面作者头像和称号下方，显示作者标签。

### 编辑方式

编辑 `configs/about/author-tags.md`：

```markdown
---
title: "作者标签"
order: 4
show: true
---

# 作者标签
# 每行一个标签，以 # 开头的行为注释

青年开发者
开源爱好者
技术分享者
全栈开发者
```

### 显示位置

```
┌─────────────────────────────┐
│        [头像]               │
│       作者名称              │
│       称号/角色              │
│   [标签1] [标签2] [标签3]   │  ← 作者标签在这里
└─────────────────────────────┘
```

### 隐藏作者标签

将 `author-tags.md` 中的 `show` 设为 `false`：

```markdown
---
title: "作者标签"
order: 4
show: false
---
```

或在 `configs/about.json` 中设置：

```json
{
  "authorTags": {
    "enabled": false,
    "tags": []
  }
}
```

---

## 5️⃣ 添加自定义内容区块

在 `configs/about/` 目录下创建任意 `.md` 文件即可添加新的内容区块：

```markdown
---
title: "我的项目"
order: 5
show: true
---

这里是我的项目介绍...

支持完整的 **Markdown** 格式！
```

文件名将成为配置的 key（如 `my-projects.md` → `aboutContent.myProjects`）。

---

## 📁 文件结构

```
configs/about/
├── about-me.md        ← 关于我
├── about-blog.md      ← 关于博客
├── tech-stack.md      ← 技术栈
└── author-tags.md     ← 作者标签
```

修改后需要重新构建才能生效：

```bash
node scripts/build-about.js
npm run build
```

---

## ❓ 常见问题

### Q: 修改后没有生效？
A: 运行 `node scripts/build-about.js` 重新生成配置，然后重新部署。

### Q: 可以同时使用 MD 和 JSON 吗？
A: 可以，MD 文件优先级更高。如果 MD 文件不存在，会回退到 JSON 配置。

### Q: 支持 Markdown 格式吗？
A: 支持！`about-me.md` 和 `about-blog.md` 的内容会通过 MarkdownRenderer 渲染。
